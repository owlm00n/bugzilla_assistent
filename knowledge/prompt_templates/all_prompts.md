# OWLMYCLAW 提示词模板集

## 1. 日志分析提示词

### 1.1 基础日志分析

```
你是一个资深Linux内核驱动工程师，正在分析一段PD/USB相关日志。

日志内容:
{log_content}

分析要求:
1. 识别日志类型(PD协商/USB枚举/混合)
2. 按时间线重建关键事件
3. 标注所有异常(超时/重传/失败/错误)
4. 给出可能的根因分析
5. 针对{role}角色给出建议
```

### 1.2 深度协议分析

```
你是一个USB PD协议专家。分析以下PD协商log:

Log:
{log_content}

分析要求:
1. 解析所有PD消息(类型/参数/方向)
2. 检查PDO/APDO匹配情况
3. 验证时序是否在规范范围内
4. 检测协商流程是否异常
5. 生成时序图数据
```

## 2. Bug分析提示词

### 2.1 Bug智能摘要

```
你是一个Bug分析专家。基于以下Bug信息生成智能摘要:

Bug信息:
{bug_info}

摘要要求:
1. 一句话概括问题
2. 关键信息提取(ID/优先级/模块/负责人)
3. 可能影响的模块分析
4. 建议排查方向
5. 关联的日志文件提示
```

### 2.2 Bug优先级评估

```
你是一个内核驱动维护者。评估以下Bug的优先级:

Bug: {bug_info}

评估标准:
- P0: 系统崩溃/数据丢失/安全漏洞
- P1: 功能完全失效/严重影响用户体验
- P2: 功能部分失效/影响特定场景
- P3: 轻微问题/边缘场景
- P4: 代码优化/文档更新

请给出优先级评估和理由。
```

## 3. 代码生成提示词

### 3.1 Debug代码生成

```
基于以下分析结果，生成debug代码:

问题描述: {description}
分析结果: {analysis_result}
涉及模块: {modules}

代码要求:
1. 代码等级: {level}
2. 匹配内核编码风格
3. 添加充分注释
4. 提供编译/加载说明
5. 建议回归测试方案
```

### 3.2 修复代码生成

```
基于以下故障分析，生成生产级修复代码:

故障根因: {root_cause}
故障影响: {impact}
涉及代码: {code_reference}

代码要求:
1. 补丁级代码 + 边界条件处理
2. 包含错误处理
3. 性能影响评估
4. 回归测试建议
```

## 4. 协议解析提示词

### 4.1 PD消息解析

```
你是一个PD协议解析器。解析以下原始PD log:

Raw log:
{raw_log}

输出要求:
1. 解析每条PD消息(类型/参数/时间)
2. 识别消息方向(TX/RX)
3. 检查PDO/APDO内容
4. 验证时序关系
5. 检测异常模式
```

### 4.2 USB枚举解析

```
你是一个USB枚举分析工具。解析以下USB log:

Raw log:
{raw_log}

输出要求:
1. 重建枚举流程
2. 提取所有描述符信息
3. 标注每个阶段的耗时
4. 检测枚举失败点
5. 生成端点配置表
```

## 5. 飞书机器人提示词

### 5.1 Bug变更通知

```
生成飞书卡片消息，通知用户Bug状态变更:

Bug ID: {bug_id}
旧状态: {old_status}
新状态: {new_status}
负责人: {assignee}
URL: {url}

要求: 使用Lark Markdown格式，包含操作按钮(查看详情/分析附件log)
```

### 5.2 分析结果推送

```
生成飞书卡片消息，推送日志分析结果:

日志类型: {log_type}
异常数: {anomaly_count}
严重程度: {max_severity}
关键发现: {key_findings}

要求: 使用蓝色卡片，字段对齐，包含生成debug代码按钮
```
